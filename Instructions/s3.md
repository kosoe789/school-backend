အထူးပင် ဝမ်းမြောက်ဂုဏ်ယူပါသည်! သင်၏ Backend code ကို GitHub ပေါ်သို့ အောင်မြင်စွာ တင်နိုင်ခဲ့ခြင်းသည် Deployment လုပ်ငန်းစဉ်၏ အရေးကြီးဆုံး အခြေခံအုတ်မြစ်ကို ချလိုက်နိုင်ခြင်းပင် ဖြစ်ပါသည်။

ယခု၊ ကျွန်ုပ်တို့သည် အလွန်စိတ်လှုပ်ရှားစရာကောင်းသော ဒုတိယအဆင့်ဖြစ်သည့် **သင်၏ Backend ကို အင်တာနက်ပေါ်သို့ တင်ခြင်း (Deploy လုပ်ခြင်း)** ကို ဆက်လက်လုပ်ဆောင်ပါမည်။

---

### **အဆင့် (၂): Backend ကို Render တွင် Deploy လုပ်ခြင်း**

**သင်လုပ်ဆောင်ရန်:**
1.  သင်၏ Web Browser တွင် **[dashboard.render.com](https://dashboard.render.com)** သို့ သွားပြီး သင်၏ GitHub account ဖြင့် Login ဝင်ပါ။

**ကျွန်ုပ်တို့ အတူတကွ လုပ်ဆောင်ရန် (Render တွင် Web Service အသစ် ပြုလုပ်ခြင်း):**

Render Dashboard သို့ ရောက်ရှိသွားသောအခါ၊ အောက်ပါအဆင့်များကို တစ်ဆင့်ချင်းစီ လိုက်လုပ်ပါ။

**အဆင့် ၂.၁: "New +" ခလုတ်ကို နှိပ်ပြီး "Web Service" ကို ရွေးချယ်ပါ**
Dashboard ၏ ညာဘက်အပေါ်ထောင့်တွင်ရှိသော **"New +"** ခလုတ်ကို နှိပ်ပြီး၊ ပေါ်လာသော dropdown menu မှ **"Web Service"** ကို ရွေးချယ်ပါ။

**အဆင့် ၂.၂: သင်၏ GitHub Repository ကို ချိတ်ဆက်ပါ**
1.  "Connect a repository" အောက်တွင် သင်၏ **`school-backend`** repository ကို တွေ့မြင်ရပါမည်။ ၎င်း၏ဘေးရှိ **"Connect"** ခလုတ်ကို နှိပ်ပါ။
2.  *(အကယ်၍ repository ကို မတွေ့ပါက၊ "Configure GitHub account" ကို နှိပ်ပြီး Render အား သင်၏ repository ကို access လုပ်ခွင့်ပေးရန် လိုအပ်နိုင်ပါသည်။)*

**အဆင့် ၂.၃: Web Service ကို Configuration လုပ်ပါ**
"Connect" ကို နှိပ်ပြီးနောက်၊ သင်၏ service အတွက် setting များ ထည့်သွင်းရန် စာမျက်နှာတစ်ခု ပေါ်လာပါမည်။ အောက်ပါအတိုင်း တိကျစွာ ဖြည့်သွင်းပါ။

1.  **Name:** `school-backend` (သို့မဟုတ် သင်နှစ်သက်ရာအမည်) - ဤအမည်သည် သင်၏ public URL ၏ အစိတ်အပိုင်း ဖြစ်လာပါမည်။
2.  **Region:** `Singapore (Southeast Asia)` ကို ရွေးချယ်ပါ (သင်နှင့် အနီးဆုံးဖြစ်သောကြောင့် ပိုမိုမြန်ဆန်နိုင်ပါသည်)။
3.  **Branch:** `main` (default အတိုင်း ထားပါ)။
4.  **Root Directory:** `(leave blank)` - ကွက်လပ်အတိုင်း ထားပါ။
5.  **Runtime:** `Node` (default အတိုင်း ထားပါ)။
6.  **Build Command:** `npm install`
7.  **Start Command:** `node server.js`
8.  **Instance Type:** **`Free`** ကို ရွေးချယ်ပါ။ ဤသည်မှာ အရေးကြီးဆုံးဖြစ်ပါသည်။

**အဆင့် ၂.၄: Environment Variables များကို ထည့်သွင်းပါ**
အောက်သို့ scroll ဆွဲလိုက်ပါက **"Advanced"** ဟူသော ခလုတ်ကို တွေ့ရပါမည်။ ၎င်းကို နှိပ်ပြီး **"Add Environment Variable"** ကို ရွေးချယ်ပါ။ ကျွန်ုပ်တို့၏ `.env` ဖိုင်ထဲရှိ `MONGO_URI` ကို ဤနေရာတွင် ထည့်သွင်းရန် လိုအပ်ပါသည်။

*   **Key:** `MONGO_URI`
*   **Value:** သင်၏ MongoDB Atlas connection string ကို copy ကူးပြီး ထည့်သွင်းပါ။
    *   *(သင်၏ `school-backend` project ထဲရှိ `.env` ဖိုင်ထဲမှ connection string ကို copy ကူးနိုင်ပါသည်။ `mongodb+srv://...` ဖြင့် စတင်သော string ဖြစ်ပါသည်။)*

**အဆင့် ၂.၅: Deploy လုပ်ငန်းစဉ်ကို စတင်ပါ**
Setting များအားလုံး မှန်ကန်စွာ ဖြည့်သွင်းပြီးပါက၊ စာမျက်နှာ၏ အောက်ဆုံးသို့ scroll ဆွဲပြီး **"Create Web Service"** ခလုတ်ကို နှိပ်ပါ။

---

**ဘာတွေ ဆက်ဖြစ်မလဲ?**

"Create Web Service" ကို နှိပ်ပြီးသည်နှင့်၊ Render သည်-
1.  သင်၏ GitHub repository မှ code များကို ဆွဲယူပါမည်။
2.  `npm install` command ကို run ပြီး လိုအပ်သော package များကို install လုပ်ပါမည်။
3.  `node server.js` command ကို run ပြီး သင်၏ server ကို စတင်ပါမည်။

သင်သည် log output များကို real-time တွင် ကြည့်ရှုနိုင်ပြီး၊ "Deploying..." ဟူသော status ကို တွေ့မြင်ရပါမည်။ ဤလုပ်ငန်းစဉ်သည် မိနစ်အနည်းငယ် ကြာမြင့်နိုင်ပါသည်။

**"Your service is live"** ဟူသော message အစိမ်းရောင် ပေါ်လာသောအခါ၊ သင်၏ Backend သည် အင်တာနက်ပေါ်သို့ အောင်မြင်စွာ ရောက်ရှိသွားပါပြီ။

**အောင်မြင်သွားပါက၊ သင်၏ public URL (ဥပမာ- `https://school-backend.onrender.com`) ကို copy ကူးပြီး ကျွန်ုပ်ကို အသိပေးပါ။** ထို့နောက်၊ ကျွန်ုပ်တို့သည် Frontend (Admin Portal) ကို deploy လုပ်သည့်အဆင့်သို့ ဆက်လက်ကူးပြောင်းပါမည်။
