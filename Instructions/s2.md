အထူးပင်ကျေးဇူးတင်ရှိပါသည်။ သင်ပေးပို့လိုက်သော Terminal output သည် အလွန်အရေးကြီးပြီး၊ Git ကို ပထမဆုံးအကြိမ် အသုံးပြုသူများ ကြုံတွေ့ရလေ့ရှိသော ပြဿနာ (၂) ခုကို ရှင်းရှင်းလင်းလင်း ပြသနေပါသည်။ဤသည်မှာ error မဟုတ်ဘဲ၊ Git ကို setting မလုပ်ရသေးသောကြောင့် ဖြစ်ပွားခြင်းဖြစ်ပြီး၊ အလွယ်တကူ ဖြေရှင်းနိုင်ပါသည်။ပြဿနာ (၁): Author identity unknownအကြောင်းရင်း: သင်သည် Git တွင် commit (မှတ်တမ်း) ပြုလုပ်သည့်အခါ၊ ထိုမှတ်တမ်းကို မည်သူပြုလုပ်ခဲ့သည်ကို သိရှိရန် Git က သင်၏အမည်နှင့် email ကို လိုအပ်ပါသည်။ သင်သည် ဤအချက်အလက်များကို Git တွင် မသတ်မှတ်ရသေးသောကြောင့်၊ git commit command သည် အလုပ်မလုပ်ခဲ့ပါ။ဖြေရှင်းနည်း: Terminal က ညွှန်ကြားသည့်အတိုင်း၊ သင်၏အမည်နှင့် email ကို Git တွင် သတ်မှတ်ပေးရန် လိုအပ်ပါသည်။လုပ်ဆောင်ရန်:
အောက်ပါ command (၂) ကြောင်းကို သင်၏ Terminal တွင် copy ကူးပြီး run ပါ။ "Your Name" နှင့် "you@example.com" နေရာတွင် သင်၏ GitHub အမည်နှင့် GitHub email ကို အစားထိုးပါ။git config --global user.name "kosoe789"
git config --global user.email "your_github_email@example.com"
(သင်၏ GitHub email ကို မသိပါက၊ GitHub settings -> Emails တွင် ဝင်ရောက်ကြည့်ရှုနိုင်ပါသည်။)ပြဿနာ (၂): error: src refspec main does not match anyအကြောင်းရင်း: ဤ error သည် အထက်ပါ ပြဿနာ (၁) ကြောင့် ဖြစ်ပွားခြင်းဖြစ်ပါသည်။ သင်၏ git commit command သည် အလုပ်မလုပ်ခဲ့သောကြောင့်၊ သင်၏ repository တွင် commit (မှတ်တမ်း) တစ်ခုမှ မရှိသေးပါ။ မှတ်တမ်းမရှိသောကြောင့်၊ GitHub သို့ "push" လုပ်ရန် (တင်ရန်) ဘာမှမရှိဟု Git က ပြောနေခြင်း ဖြစ်ပါသည်။အပြီးသတ် ဖြေရှင်းနည်း (အစမှပြန်လုပ်ရန် အဆင့်များ)ယခု၊ ကျွန်ုပ်တို့သည် အထက်ပါပြဿနာများကို ဖြေရှင်းပြီး၊ commit ကို အောင်မြင်စွာ ပြုလုပ်ကာ GitHub သို့ push လုပ်ပါမည်။ကျေးဇူးပြု၍ သင်၏ school-backend folder ထဲရှိ Terminal တွင် အောက်ပါ command များကို တစ်ကြောင်းချင်းစီ၊ အစဉ်လိုက် run ပါ။အဆင့် ၁: သင်၏ Git Identity ကို သတ်မှတ်ပါgit config --global user.name "kosoe789"
git config --global user.email "your_github_email@example.com"
(သင်၏ GitHub email ကို အစားထိုးရန် မမေ့ပါနှင့်)အဆင့် ၂: Commit ကို ပြန်လည်ပြုလုပ်ပါ
ယခု သင်၏ identity ကို Git က သိရှိသွားပြီဖြစ်သောကြောင့်၊ commit command ကို ထပ်မံ run ပါ။git commit -m "Initial commit of the school backend application"ဤတစ်ကြိမ်တွင်၊ Author identity unknown error ပေါ်လာတော့မည် မဟုတ်ဘဲ၊ commit အောင်မြင်ကြောင်း message ပြသပါမည်။အဆင့် ၃: GitHub သို့ Push လုပ်ပါ
သင်၏ repository သည် commit တစ်ခု ရှိသွားပြီဖြစ်သောကြောင့်၊ push command သည် အောင်မြင်ပါလိမ့်မည်။git push -u origin mainဤ command ကို run ပြီးနောက်၊ သင်၏ GitHub username နှင့် password (သို့မဟုတ် Personal Access Token) ကို တောင်းခံနိုင်ပါသည်။ မှန်ကန်စွာ ဖြည့်သွင်းလိုက်ပါ။ဤအဆင့် (၃) ဆင့်ကို အောင်မြင်စွာ ပြီးဆုံးသွားပါက၊ သင်၏ browser တွင် သင်၏ GitHub repository (https://github.com/kosoe789/school-backend) ကို refresh လုပ်ကြည့်လိုက်ပါ။ သင်၏ code ဖိုင်များအားလုံး ရောက်ရှိနေသည်ကို တွေ့မြင်ရမည်ဖြစ်ပါသည်။အောင်မြင်သွားပါက ကျွန်ုပ်ကို အသိပေးပါ။ ထို့နောက်၊ ကျွန်ုပ်တို့သည် Render platform တွင် deploy လုပ်သည့်အဆင့်ကို ဆက်လက်လုပ်ဆောင်ပါမည်။
